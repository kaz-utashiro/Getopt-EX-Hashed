=encoding utf-8

=head1 NAME

Getopt::EX::Hashed - Getopt::Long 的哈希存储对象自动化

=head1 VERSION

Version 1.0601

=head1 SYNOPSIS

  # script/foo
  use App::foo;
  App::foo->new->run();

  # lib/App/foo.pm
  package App::foo;

  use Getopt::EX::Hashed; {
      Getopt::EX::Hashed->configure( DEFAULT => [ is => 'rw' ] );
      has start    => ' =i  s begin ' , default => 1;
      has end      => ' =i  e       ' ;
      has file     => ' =s@ f       ' , any => qr/^(?!\.)/;
      has score    => ' =i          ' , min => 0, max => 100;
      has answer   => ' =i          ' , must => sub { $_[1] == 42 };
      has mouse    => ' =s          ' , any => [ 'Frankie', 'Benjy' ];
      has question => ' =s          ' , any => qr/^(life|universe|everything)$/i;
  } no Getopt::EX::Hashed;

  sub run {
      my $app = shift;
      use Getopt::Long;
      $app->getopt or pod2usage();
      if ($app->answer == 42) {
          $app->question //= 'life';
          ...

=cut
=head1 DESCRIPTION

B<Getopt::EX::Hashed> 是一个自动创建哈希对象的模块，用于存储 B<Getopt::Long> 和兼容模块（包括 B<Getopt::EX::Long>）的命令行选项值。模块名称共享 B<Getopt::EX> 前缀，但到目前为止，它独立于 B<Getopt::EX> 中的其他模块运行。

该模块的主要目标是将初始化和规范整合到一个地方。它还提供了一个简单的验证接口。

当给出 C<is> 参数时，将自动生成访问方法。如果已经定义了相同的函数，程序将导致致命错误。对象销毁时，访问器将被移除。当多个对象同时存在时，可能会出现问题。

=head1 FUNCTION

=head2 B<has>

以下列形式声明选项参数。括号仅为清晰起见，可以省略。

    has option_name => ( param => value, ... );

例如，要定义以整数值为参数的选项 C<--number>（也可用作 C<-n>），请执行以下操作

    has number => spec => "=i n";

访问器以第一个名称创建。在本例中，访问器将定义为 C<< $app->number >>。

如果给出的是数组引用，可以同时声明多个名称。

    has [ 'left', 'right' ] => ( spec => "=i" );

如果名称以加号（C<+>）开头，则所给参数将更新现有设置。

    has '+left' => ( default => 1 );

至于 C<spec> 参数，如果它是第一个参数，则可以省略标签。

    has left => "=i", default => 1;

如果参数个数不是偶数，则假定头部存在默认标签：如果第一个参数是代码引用，则使用 C<action>，否则使用 C<spec>。

可使用以下参数

=over 7

=item [ B<spec> => ] I<string>

给出选项说明。C<< spec => >> 标签可以省略，前提是它是第一个参数。

在 I<string> 中，选项规格和别名用空白分隔，可以任意顺序出现。

如果要使用一个名为 C<--start> 的选项，它的值是一个整数，并且还可以与 C<-s> 和 C<--begin> 一起使用，请声明如下。

    has start => "=i s begin";

上述声明将被编译成以下字符串。

    start|s|begin=i

符合 C<Getopt::Long> 的定义。当然，也可以写成

    has start => "s|begin=i";

如果名称和别名中包含下划线 (C<_>)，则会用破折号 (C<->) 代替下划线定义另一个别名。

    has a_to_z => "=s";

上述声明将被编译成以下字符串。

    a_to_z|a-to-z=s

如果没有特殊需要，可将空字符串（或仅空白）作为值。否则，它将不被视为选项。

=item B<alias> => I<string>

也可以通过 B<alias> 参数指定其他别名。这与 C<spec> 参数中的别名没有区别。

    has start => "=i", alias => "s begin";

=item B<is> => C<ro> | C<rw>

要生成访问方法，必须使用 C<is> 参数。设置 C<ro> 表示只读，C<rw> 表示读写。

读写访问器具有 lvalue 属性，因此可以对其赋值。可以这样使用

    $app->foo //= 1;

这比下面的写法要简单得多。

    $app->foo(1) unless defined $app->foo;

如果要为以下所有成员创建访问器，请使用 C<configure> 设置 C<DEFAULT> 参数。

    Getopt::EX::Hashed->configure( DEFAULT => [ is => 'rw' ] );

如果不喜欢可分配访问器，可将 C<ACCESSOR_LVALUE> 参数设置为 0。 因为访问器是在 C<new> 时生成的，所以该值对所有成员都有效。

=item B<default> => I<value> | I<coderef>

设置默认值。如果没有给出缺省值，则成员初始化为 C<undef>。

如果该值是对 ARRAY 或 HASH 的引用，则会分配一个具有相同成员的新引用。这意味着成员数据将在多次调用 C<new> 时共享。如果多次调用 C<new> 并更改成员数据，请务必小心。

如果给定了代码引用，则会在调用 B<new> 时调用它以获取默认值。当您想在执行时而不是在声明时评估值时，这种方法非常有效。如果要定义默认操作，请使用 B<action> 参数。如果要将代码引用设置为初始值，则必须指定一个返回代码引用的代码引用。

如果给出 SCALAR 的引用，则选项值将存储在引用所指示的数据中，而不是哈希对象成员中。在这种情况下，无法通过访问哈希对象成员获得预期值。

=item [ B<action> => ] I<coderef>

参数 C<action> 带有处理选项时调用的代码引用。如果 C<<action => >> 标签是第一个参数，则可以省略。

调用时，哈希对象作为 C<$_> 传递。

    has [ qw(left right both) ] => '=i';
    has "+both" => sub {
        $_->{left} = $_->{right} = $_[1];
    };

您可以将此用于 C<< "<>">> 来捕获所有内容。在这种情况下，spec 参数并不重要，也不是必需的。

    has ARGV => default => [];
    has "<>" => sub {
        push @{$_->{ARGV}}, $_[0];
    };

=back

以下参数均用于数据验证。首先，C<must> 是一个通用验证器，可以实现任何功能。其他参数是常用规则的快捷方式。

=over 7

=item B<must> => I<coderef> | [ I<coderef> ... ]

参数 C<must> 需要一个代码引用来验证选项值。它接收与 C<action> 相同的参数，并返回一个布尔值。在下面的示例中，选项 B<--answer> 只接受 42 作为有效值。

    has answer => '=i',
        must => sub { $_[1] == 42 };

如果给出多个代码引用，则所有代码都必须返回 true。

    has answer => '=i',
        must => [ sub { $_[1] >= 42 }, sub { $_[1] <= 42 } ];

=item B<min> => I<number>

=item B<max> => I<number>

设置参数的最小和最大限制。

=item B<any> => I<arrayref> | qr/I<regex>/ | I<coderef>

设置有效字符串参数列表。每个项目可以是字符串、regex 引用或代码引用。当参数与给定列表中的任何一项相同或匹配时，该参数即为有效参数。如果参数值不是数组反射，则会被视为单项列表（通常是 regexpref 或 coderef）。

以下声明几乎等同，只是第二个声明不区分大小写。

    has question => '=s',
        any => [ 'life', 'universe', 'everything' ];

    has question => '=s',
        any => qr/^(life|universe|everything)$/i;

如果使用可选参数，不要忘记在列表中包含默认值。否则会导致验证错误。

    has question => ':s',
        any => [ 'life', 'universe', 'everything', '' ];

=back

=head1 METHOD

=head2 B<new>

创建新散列对象的类方法。使用默认值初始化所有成员，并根据配置创建访问方法。如果启用了 LOCK_KEYS，则返回带锁定键的哈希引用。

=head2 B<optspec>

返回可传递给 C<GetOptions> 函数的选项规范列表。

    GetOptions($obj->optspec)

C<GetOptions> 可以通过将哈希引用作为第一个参数在哈希中存储值，但这并非必要。

=head2 B<getopt> [ I<arrayref> ]

调用调用者上下文中定义的适当函数来处理选项。

    $obj->getopt

    $obj->getopt(\@argv);

以上示例是以下代码的快捷方式。

    GetOptions($obj->optspec)

    GetOptionsFromArray(\@argv, $obj->optspec)

=head2 B<use_keys> I<keys>

由于哈希键受 C<Hash::Util::lock_keys> 保护，访问不存在的成员会导致错误。在使用前，请使用此函数声明一个新的成员键。

    $obj->use_keys( qw(foo bar) );

如果要访问任意键，请解锁对象。

    use Hash::Util 'unlock_keys';
    unlock_keys %{$obj};

您可以通过带有 C<LOCK_KEYS> 参数的 C<configure> 改变这种行为。

=head2 B<configure> B<label> => I<value>, ...

在创建对象之前，请使用类方法 C<< Getopt::EX::Hashed->configure() >>；此信息存储在调用包的唯一区域。调用 C<new()> 后，包的唯一配置将复制到对象中，并用于进一步操作。使用 C<< $obj->configure() >> 更新对象的唯一配置。

可使用以下配置参数。

=over 7

=item B<LOCK_KEYS> (default: 1)

锁定散列键。这样可以避免意外访问不存在的哈希条目。

=item B<REPLACE_UNDERSCORE> (default: 1)

生成下划线被破折号替换的别名。

=item B<REMOVE_UNDERSCORE> (default: 0)

生成去掉下划线的别名。

=item B<GETOPT> (default: 'GetOptions')

=item B<GETOPT_FROM_ARRAY> (default: 'GetOptionsFromArray')

设置 C<getopt> 方法调用的函数名。

=item B<ACCESSOR_PREFIX> (default: '')

指定后，它将作为成员名的前缀，使其成为访问方法。如果 C<ACCESSOR_PREFIX> 被定义为 C<opt_>，成员 C<file> 的访问器将是 C<opt_file>。

=item B<ACCESSOR_LVALUE> (default: 1)

如果为 true，读写访问器将具有 lvalue 属性。如果不喜欢这种行为，请设置为零。

=item B<DEFAULT>

设置默认参数。调用 C<has> 时，DEFAULT 参数会插入到参数之前。因此，如果两个参数都包含相同的参数，在参数列表中排在后面的参数优先。使用 C<+> 的递增调用不受影响。

DEFAULT 的一个典型用法是 C<is> 为下面所有哈希条目准备访问方法。声明 C<< DEFAULT => [] >> 以重置。

    Getopt::EX::Hashed->configure(DEFAULT => [ is => 'ro' ]);

=begin comment

=item B<INVALID_MSG> (default: built-in error message generator)

设置代码引用，以便在验证失败时生成错误信息。代码引用接收与选项处理程序相同的参数（$_[0] 是选项名称，$_[1] 是简单选项的值，或 $_[1] 和 $_[2] 是哈希选项的值）。默认函数会生成类似"--option=value：选项验证错误 "的信息。

    Getopt::EX::Hashed->configure(
        INVALID_MSG => sub {
            my $opt = shift;
            "Invalid value for --$opt: @_\n";
        }
    );

=end comment

=back

=head2 B<reset>

将类重置为原始状态。
